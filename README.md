# Playwright com Copilot: MCP, Skills e TDD

_Aprenda, em menos de uma hora, a escrever testes automatizados de ponta a ponta com **Playwright**, guiado pelo **GitHub Copilot** usando **MCP**, **Skills** (instruções customizadas) e o ciclo **TDD** (Red → Green → Refactor)._

## Bem-vindo(a) 👋

- **Para quem é:** pessoas iniciantes em automação de testes que já programam um pouquinho e querem ver como o Copilot pode ajudar a escrever testes E2E confiáveis.
- **O que você vai aprender:**
  1. Como conectar o **Playwright MCP** ao Copilot dentro do VS Code.
  2. Como criar um arquivo `.github/copilot-instructions.md` (uma “Skill” do projeto) para o Copilot seguir suas convenções.
  3. Como aplicar **TDD** (escrever o teste antes do código) usando o Copilot como par de programação.
- **O que você vai construir:** uma mini página estática (uma landing page de newsletter) com testes Playwright cobrindo o fluxo de inscrição.
- **Pré-requisitos:**
  - Conta no GitHub com **Copilot** habilitado (a versão Free funciona).
  - Familiaridade básica com [VS Code](https://code.visualstudio.com/) e linha de comando.
  - Recomendado: o exercício [Introduction to GitHub](https://github.com/skills/introduction-to-github).
- **Quanto tempo:** menos de uma hora.

Neste exercício você vai:

1. Abrir um **Codespace** já configurado com Node.js e Playwright.
2. Ativar o **Playwright MCP** e criar um arquivo de **instruções do Copilot** (Skill do projeto).
3. Aplicar **TDD**: escrever um teste que **falha** primeiro (RED).
4. Implementar a feature com a ajuda do **Copilot Agent + MCP** até o teste **passar** (GREEN).
5. **Refatorar** e rodar os testes em **GitHub Actions** (CI).
